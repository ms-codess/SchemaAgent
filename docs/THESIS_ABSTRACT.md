# Thesis Abstract

Text-to-SQL systems often fail in enterprise settings because the information needed to write a correct query is not fully contained in the relational schema. Column names can be ambiguous, encoded values may not be self-explanatory, and key business rules frequently exist only in external documentation. This thesis presents SchemaAgent, a hybrid Text-to-SQL architecture that combines semantic schema retrieval, an agentic self-correction loop, and document retrieval over auxiliary knowledge sources. On the BIRD Mini-Dev benchmark, the main system configuration achieves 63.2% execution accuracy, outperforming zero-shot prompting and single-component ablations. The thesis then introduces a document-grounding experiment in which BIRD's gold evidence hints are removed and replaced with retrieval over `database_description/*.csv` files, treated as enterprise-style data dictionaries. This evaluates whether retrieval over documentation can recover domain knowledge that would otherwise be unavailable at inference time. The central claim is that robust enterprise Text-to-SQL requires a layered grounding strategy: schema retrieval constrains the structural search space, self-correction repairs execution-time failures, and document retrieval supplies the semantic context needed to bridge the gap between database structure and business meaning.
